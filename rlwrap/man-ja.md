# rlwrap(1) — 一般コマンドリファレンス

## 名前

**rlwrap** — readline ラッパー

## 書式

```
rlwrap [rlwrap-options] command ...
```

## 説明

`rlwrap` は指定したコマンドを実行し、ユーザーの入力に割り込むことで readline の行編集・永続ヒストリ・補完機能を提供します。

`rlwrap` は完全に透過的に動作することを目指しており（ほぼ成功しています）、`command` と `rlwrap command` の違いをユーザーやシェルが感じることはないはずです（readline 機能が追加されている点を除いて）。リダイレクト、パイプ、シグナルの送受信時や、コマンドが端末設定や作業ディレクトリを変更する場合でも同様です。

補完のカスタマイズ、複数行入力、プロンプトの色付け・書き換えなど多くのオプションがありますが、通常は必要ありません。

---

## オプション

**`-a, --always-readline [password_prompt]`**
コマンドの端末設定に関わらず、常に「readline モード」を維持します。すでに行編集を行うコマンドに対して `rlwrap` を使いたい場合に使用します。注意: このオプションを使うと、パスワードプロンプトを引数として指定しない限り、パスワードがエコー表示・保存されます。引数は省略可能で、指定する場合はオプションの直後にスペースなしで続けてください。

Linux では `-N (--no-children)` オプションを組み合わせることで、コマンドから呼び出されるページャやエディタをラップしないようにでき、より使いやすくなります。

`--always-readline` を必要とするコマンドの多くは、端末制御シーケンスが `rlwrap` を混乱させないよう `-t dumb` も必要な場合があります（ただしページャ・エディタには不便です）。

**`-A, --ansi-colour-aware`**
カラープロンプトを使うコマンドは `rlwrap` を混乱させることがあります。このオプションでその挙動を改善します。プロンプトに ANSI カラーコード以上の複雑なエスケープシーケンスが含まれる場合は逆効果になることがあります。

**`-b, --break-chars list_of_characters`**
指定した文字を単語区切り文字として扱います（空白は常に区切り文字）。補完時および `-f` オプションで指定したファイルから補完候補リストを作成する際の「単語」の定義に影響します。デフォルト: `(){}[],'+-=&^%$#@"";|\`。`-c` を指定しない場合、`/` と `.`（ピリオド）もデフォルトリストに含まれます。

**`-c, --complete-filenames`**
ファイル名補完を有効にします（ファイル名補完は `-i` オプション指定時でも常に大文字小文字を区別します）。Linux・OS X・FreeBSD・Solaris では、コマンドの作業ディレクトリを追跡するため、相対パスの補完が期待通りに動作します。

**`-C, --command-name command_name|N`**
ヒストリ・補完ファイルの名前の決定と readline の初期化（`~/.inputrc` に基づく）に、`command` の代わりに `command_name` を使用します。数値 N (>0) を指定すると、引数リストの末尾から N 番目の引数を使用します。

**`-D, --history-no-dupes n`**
入力ヒストリから重複エントリを削除する強度を指定します。`n=0`: すべて保持、`n=1`（デフォルト）: 連続する重複を削除、`n=2`: 同じ内容の過去の全エントリを削除。

**`-e, --extra-char-after-completion char`**
補完テキスト挿入後に付加する文字を変更します（デフォルトはスペース）。`''` で何も付加しないようにできます。

**`-f, --file file`**
ファイルを単語に分割して補完候補リストに追加します。複数回指定可能。`-f .` とすると現在のヒストリファイルを補完候補として使用します。

**`-g, --forget-matching regexp`**
指定した POSIX 正規表現にマッチする入力行をヒストリに追加しません。マッチは常に大文字小文字を無視します。

**`-h, --help`**
簡易ヘルプを表示します。

**`-H, --history-filename file`**
指定したファイルからコマンドヒストリを読み込み（`--histsize >= 0` の場合は書き戻しも行います）。

**`-i, --case-insensitive`**
補完時に大文字小文字を無視します（ファイル名補完は除く）。このオプションは `-f` オプションより前に指定する必要があります。

**`-I, --pass-sigint-as-sigterm`**
INT シグナル受信時（例: CTRL-C）に、コマンドへ TERM シグナルを送信します。

**`-l, --logfile file`**
readline モード時、コマンドの出力（エコーされたユーザー入力を含む）を指定したファイルに追記します。

**`-n, --no-warnings`**
警告を表示しません。

**`-N, --no-children`**
コマンドの子プロセスをラップしません。コマンドが子プロセスを待機していることを検知すると、ダイレクトモードに切り替えてキー入力をそのまま渡します。`--always-readline` を必要とするコマンドからエディタやページャを呼び出す際に便利です。Linux カーネルで `CONFIG_KALLSYMS` が有効な場合のみ動作します。

**`-m, --multi-line [newline_substitute]`**
「改行代替」文字列（デフォルト: ` \ `〈スペース・バックスラッシュ・スペース〉）を使った複数行入力を有効にします。改行代替はコマンドへ送信前に改行に変換されます。このオプションを使うと `CTRL-^`（デフォルト）で現在の入力を外部エディタで編集できます。

**`-M, --multi-line-ext .ext`**
複数行エディタで使う一時ファイルの拡張子を指定します（例: 構文ハイライトのため）。

**`-o, --one-shot`**
最初の入力行を受け付けた後、コマンドへ EOF を送信します。

**`-O, --only-cook regexp`**
指定した正規表現にマッチするプロンプトのみを「調理（cook）」します。

**`-p, --prompt-colour [colour_name|Colour_name|colour_spec]`**
プロンプトに色を付けます。色名は `black`, `red`, `green`, `yellow`, `blue`, `cyan`, `purple`（=magenta）, `white`、または ANSI カラーコード形式（例: `-p'1;31'` で太字赤）を指定できます。大文字始まりの色名（`Yellow` や `YELLOW`）は太字になります。このオプションは `--ansi-colour-aware` を暗黙的に有効にします。

**`-P, --pre-given text`**
起動時に指定テキストを編集バッファに入れておきます（`--always-readline` が自動的に設定されます）。

**`-q, --quote-characters list_of_characters`**
括弧のマッチングなどで引用符として扱う文字を指定します（デフォルト: `"'`）。

**`-r, --remember`**
入出力に現れたすべての単語を補完リストに追加します。

**`-R, --renice`**
rlwrap の優先度をコマンドより低くします（`nice(1)` 参照）。コマンドの出力中に rlwrap が割り込むのを防ぐのに役立ちます。

**`-s, --histsize N`**
ヒストリリストを N エントリに制限します（デフォルト: 300）。負の値 `-N`（`-0` も含む）は N と同じですが、ヒストリファイルを読み取り専用として扱います。

**`-S, --substitute-prompt prompt`**
コマンドのプロンプトを指定したプロンプトで置き換えます。コマンド自体にプロンプトがない場合などに有用です。

**`-t, --set-term-name name`**
コマンドの `TERM` 環境変数を指定した名前に設定します。複雑な画面制御コードで `rlwrap` が混乱するプログラムには `-t dumb` が有効です。

**`-U, --mirror-arguments`**
（Linux のみ）`ps` コマンドで見えるコマンドの引数を追跡し、`rlwrap` 自身の引数に反映します。コマンドライン上のパスワード引数が `rlwrap` 経由で露出するのを防ぐのに主に使います。

**`-v, --version`**
rlwrap のバージョンを表示します。

**`-w, --wait-before-prompt timeout`**
コマンドの最後の出力がプロンプトかどうかを判断するために、受信後 `timeout` ミリ秒待機します。追加出力がなければプロンプトとして「調理」・表示します。タイムアウトの前にマイナス符号を付けると「patient モード」になり、調理完了後まで表示を保留します（デフォルト: 40ms）。

**`-W, --polling`**
【実験的】`-w` と同じタイムアウト間隔（デフォルト 40ms）ごとに起動し、スレーブの割り込み文字や ISIG フラグを検知して端末設定を調整します。`rlwrap` の有無で CTRL-C の挙動が異なる場合などに試してください。

**`-z, --filter filter`**
フィルタを使って `rlwrap` の動作をカスタマイズします。特定の入力をヒストリから除外したり、プロンプトを変更したり、マクロやプログラマブル補完を実装できます。Perl および Python 3 モジュールが付属しています。

```
rlwrap -z 'filter args' command ...
rlwrap -z 'pipeline filter1 ... : filter2 ... : ...' command ...
```

## 使用例

```sh
# nc（netcat）にコマンドライン編集とヒストリを追加
rlwrap nc

# smbclient（readline使用）をラップ、パスワードをヒストリに残さず、子プロセス（moreなど）はラップしない
rlwrap -aPassword: -N smbclient //PEANUT/C

# gauche（Schemeインタープリタ）を青太字プロンプト、複数行編集（.scm拡張子）、シングルクォートを引用符扱いしないでラップ
rlwrap -pBlue -m -M .scm -q'

# インストール済みフィルタの一覧
rlwrap -z listing

# sqlite3 をラップし、SQL キーワード補完（大文字小文字無視）と pipeto フィルタを有効化
rlwrap -a -z pipeto -i -f sql_words sqlite3 contacts.db

# シェルスクリプトで read の代替として使う
order=$(rlwrap -pYellow -S 'Your pizza? ' -H past_orders -P Margherita -o cat)
```

## ダイレクトモードと readline モード

コンソールコマンドは端末を「cooked モード」（ENTER まで入力を溜める）か「raw モード」（キーをすぐに渡す）で使います。`rlwrap` はコマンドを別セッションの擬似端末（pty）上で動かし、モードを監視します。raw モード時はダイレクトモード（入出力をそのまま中継）、cooked モード時は readline モード（入力を readline で編集）で動作します。

## Patient モードと Impatient モード

大量出力時、すべてのチャンクが改行で終わるとは限りません。`rlwrap` はデフォルトで 40ms 待機し、それ以上出力がなければプロンプトとして処理します（impatient モード）。`-w` に負の値を指定すると、調理完了後に表示する patient モードになります（プロンプトのちらつきが減りますが、長い出力行に干渉する場合があります）。

## プロンプトの「調理」

`rlwrap` がプロンプトを認識すると、オプションに応じてフィルタリング（`-z`）、置換（`-S`）、色付け（`-p`）の順に処理（「調理」）を行います。

## 特殊キーとバインド可能なコマンド

| キー | 動作 |
|------|------|
| `Ctrl+O` | 現在の行を実行するが、ヒストリには追加しない（`rlwrap-accept-line-and-forget`） |
| `Ctrl+^` | 外部エディタで現在の入力を編集（`-m` オプション必須）（`rlwrap-call-editor`） |
| （未バインド） | `rlwrap-direct-keypress`: そのキーをダイレクトモードでなくても常に直接コマンドへ送る |
| （未バインド） | `rlwrap-hotkey`: 現在の入力行とヒストリをフィルタに通す |
| （未バインド） | `rlwrap-hotkey-without-history`: ヒストリなしでフィルタに通す（より効率的） |

`~/.inputrc` でキーバインドを変更できます。例:

```
"\M-\C-m": rlwrap-accept-line-and-forget   # ESC+Enter でヒストリに残さず実行
"\C-xe":   rlwrap-call-editor              # Ctrl+x e でエディタ起動
$if erl
   "\C-g": rlwrap-direct-keypress          # Erlangシェル用
$endif
"\C-y": rlwrap-hotkey-without-history      # Ctrl+y でフィルタ適用
```

## 環境変数

| 変数 | 説明 |
|------|------|
| `RLWRAP_HOME` | ヒストリ・補完ファイルを保存するディレクトリ |
| `RLWRAP_EDITOR`（または `EDITOR`, `VISUAL`） | 複数行入力やヒストリ編集に使うエディタ（例: `vi +%L`）。`%L`=行番号、`%C`=列番号、`%F`=一時ファイル名 |
| `RLWRAP_FILTERDIR` | フィルタを検索するディレクトリ（デフォルト: `/usr/share/rlwrap/filters`） |

---

## シグナル

`HUP`, `INT`, `QUIT`, `USR1`, `USR2`, `TERM`, `WINCH` はコマンドへ転送されます。`TSTP`（Ctrl+Z による一時停止）も適切に処理されます。フィルタが 1 秒以上応答しない場合、Ctrl+C で中断できます（rlwrap 自体は終了します）。コマンドがシグナルで終了すると、`rlwrap` は同じシグナルで自らも終了し、シェルには同じ終了ステータスが返されます。

---

## リダイレクト

標準入力が端末でない場合、`rlwrap` はオプションを無視してコマンドをそのまま実行します。標準出力（または標準エラー）が端末でない場合、`rlwrap` はコマンド起動後に `/dev/tty` に再オープンし、コマンドの出力は期待通りにリダイレクトされつつ、キーボード入力と `rlwrap` のエラーメッセージは端末に表示されます。

---

## 終了ステータス

`rlwrap` エラー時は非ゼロ、それ以外はコマンドの終了ステータスを返します。クラッシュ時でも端末は正常な状態に戻されます。

## ファイル

| パス | 説明 |
|------|------|
| `$RLWRAP_HOME/command_history`, `~/.command_history` | コマンドのヒストリファイル |
| `$RLWRAP_HOME/command_completions`, `~/.command_completions` | ユーザーごとの補完候補リスト（rlwrap は書き込まない） |
| `/usr/share/rlwrap/completions/command` | システム共通の補完候補リスト（ユーザーファイルがない場合に使用） |
| `$INPUTRC`, `~/.inputrc` | readline の初期化ファイル |

---

## バグと制限

`rlwrap` はある意味「力技」のツールです。コマンドの内部状態を知ることができないため、コンテキスト依存の補完は不可能です。readline ライブラリをコマンド内部で直接使うのが依然として最善策です。

また、`rlwrap` はコマンドの状態と完全には同期できないため、競合状態が発生することがあります。高負荷時や「調理」・フィルタリングが多い場合に、プロンプトや出力が乱れたり色がおかしくなることがあります。制御文字を含むプロンプトの処理はうまくいかないことがあり、フィルタで事前にクリーニングすると良いでしょう。
