Tcl/Tk 8.6 は、2012年にリリースされた非常に重要なメジャーアップデートです。このバージョンでは、言語のモダン化とパフォーマンス向上、そして長年要望されていた多くの機能が標準ライブラリに取り込まれました。

主な追加機能を5つのカテゴリーに分けて要約します。

---

### 1. TclOO (オブジェクト指向プログラミングの標準搭載)
もっとも大きな変更点は、**オブジェクト指向（OO）機能がコアに統合された**ことです。
* これまで `itcl` や `xotcl` などの外部拡張が必要でしたが、標準の `oo::class` や `oo::object` コマンドでクラスベースのプログラミングが可能になりました。
* 多重継承、ミックスイン、フィルタなどの高度な機能を備えています。

### 2. コルーチン (Coroutines)
非同期処理や状態管理を簡潔に記述できる**コルーチン**が導入されました。
* `coroutine` と `yield` コマンドを使用します。
* イベントループをブロックせずに、一時停止して後で再開するような処理（例：非同期通信を同期的な見た目で書く）が容易になりました。

### 3. エラー処理とリスト操作の強化
モダンなプログラミング言語に近い制御構造が追加されました。
* **`try` と `throw`:** 従来の `catch` よりも詳細で読みやすい例外処理が可能になりました。
* **`lmap`:** リストの各要素に対して処理を行い、新しいリストを返す機能です（いわゆる `map` 関数）。`foreach` を使って手動でリストを構築する手間が省けます。

### 4. I/O とネットワーク、圧縮機能
* **IPv6 のネイティブサポート:** ネットワークソケットで IPv6 が標準で利用可能になりました。
* **zlib の統合:** `zlib` コマンドが追加され、標準機能だけで gzip 形式などの圧縮・解凍ができるようになりました。
* **スタックチャネル (Transchans):** データのストリームに対して、暗号化や変換のレイヤーを動的に重ねる機能が強化されました。

### 5. Tk (GUI) の改善
* **PNG 形式のネイティブサポート:** 外部ライブラリ（Img 等）を使わずに、標準で PNG 画像を表示・保存できるようになりました。
* **フォントのアンチエイリアス:** X11 (Linux) 等での描画品質が向上しました。
* **ビルトイン・フォント選択ダイアログ:** `tk fontchooser` が追加され、OS標準のフォント選択画面を呼び出せるようになりました。
* **GIFアニメーションの改善:** GIFの透明度処理などが向上しました。

---

### まとめ
Tcl/Tk 8.6 は、**「オブジェクト指向」と「非同期処理（コルーチン）」という現代的な武器を手に入れ、PNGやIPv6といった標準規格に追いついた**アップデートと言えます。

現在、多くのシステムで Tcl/Tk 8.6 が標準（あるいは最小要件）となっており、8.5 以前のコードを 8.6 の新機能（特に `lmap` や `try`）を使って書き直すだけで、コードの可読性は大幅に向上します。
