Tcl/Tk 9.0 は、1997年の 8.0 リリース以来、約27年ぶりとなるメジャーバージョンアップです。長年の課題であった制限の解消や、現代的なコンピューティング環境への適応が主なテーマとなっています。

主な新機能と変更点は以下の通りです。

### 1. 64ビット環境への完全対応（2GB制限の撤廃）
最も大きな内部変更です。これまでの Tcl 8.x では、文字列やリストの長さ、ファイルサイズなどが 32ビット整数（約2GB）に制限されていました。
*   **大容量データのハンドリング:** 2GB を超えるメモリ上のデータやファイルをネイティブに扱えるようになりました。
*   **内部構造の刷新:** インデックスやカウントに使用される型が `int` から `size_t` 相当に変更されました。

### 2. Unicode とエンコーディングの強化
現代的なテキスト処理に対応するため、Unicode サポートが大幅に強化されました。
*   **フル Unicode サポート:** 絵文字や特殊な漢字など、BMP（基本多言語面）以外の文字（サロゲートペアが必要だった範囲）をネイティブに正しく扱えるようになりました。
*   **デフォルトエンコーディング:** システムに関わらず、デフォルトのエンコーディングが **UTF-8** に統一されました。
*   **バイナリデータの厳格化:** 文字列とバイナリデータの区別がより明確になり、不正なバイトシーケンスの扱いが安全になりました。

### 3. Zip ファイルシステムの統合
これまで外部ライブラリや特殊なビルド（Starpackなど）が必要だった「Zipファイルの実行」が標準機能になりました。
*   **mount コマンド:** `zipfs mount` を使って、Zipアーカイブをディレクトリのようにマウントし、中のスクリプトやリソースに直接アクセスできます。
*   **配布の簡素化:** アプリケーション本体とライブラリを一つの実行ファイルにまとめるのが容易になります。

### 4. TclOO（オブジェクト指向）の拡張
標準組み込みのオブジェクト指向機能（TclOO）に、要望の多かった機能が追加されました。
*   **プライベート変数とメソッド:** クラス内部でのみ使用する変数やメソッドを定義できるようになり、カプセル化が強化されました。
*   **アクセサの自動生成:** プロパティに対する getter/setter の記述が容易になりました。

### 5. Tk（GUI）の進化
GUIツールキットである Tk も、現代的なOSの挙動に合わせるための改善が行われました。
*   **通知機能 (SysTray):** システムトレイ（タスクバーの通知領域）へのアイコン表示や、OS標準の通知ポップアップを表示する機能が追加されました。
*   **スケーリング（HiDPI）の改善:** 高解像度ディスプレイ（4K/5Kなど）での表示がより綺麗で正確になりました。
*   **SVG サポートの標準化:** 外部ライブラリなしでベクトル画像（SVG）を扱えるようになり、解像度に依存しないアイコン作成が容易になりました。
*   **新しいウィジェット/オプション:** `tk_messageBox` の改善や、より柔軟なプラットフォーム固有のテーマ設定が可能です。

### 6. 新しいコマンドと構文の改善
*   **`string insert`:** 文字列の指定位置に別の文字列を挿入するコマンドが追加されました。
*   **`tcl::process`:** 子プロセスの管理（バックグラウンドプロセスの監視や終了待ち）を行うための新しいアンサンブルコマンドです。
*   **`const`:** (検討されていた機能の一部として) 定数の概念や、パフォーマンス最適化のための内部ヒントが強化されています。

### 7. 古い機能の整理（クリーンアップ）
メジャーアップデートに伴い、古い負の遺産が削除されました。
*   Windows 95/98/XP などの古いOSのサポート終了。
*   非常に古い C API の廃止。
*   これにより、コアコードが軽量化され、将来のメンテナンス性が向上しました。

### 開発者にとっての影響
*   **移行の注意点:** 多くの Tcl 8.x スクリプトはそのまま動きますが、**「2GB以上のデータを扱う際の挙動」**や**「エンコーディング（UTF-8）のデフォルト化」**によって、バイナリデータを扱っている古いコードでは修正が必要になる場合があります。
*   **C拡張モジュール:** C言語で書かれた拡張ライブラリは、64ビット対応（`Tcl_Size` 型への変更など）のために再コンパイルやコードの修正が必要になります。

---

**まとめ:**
Tcl/Tk 9.0 は、「古い使い勝手はそのままに、現代の巨大なデータと多様な文字、高解像度なディスプレイに正式対応した」堅実かつ重要なアップデートと言えます。
